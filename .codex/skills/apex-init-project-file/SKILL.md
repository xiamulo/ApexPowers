# Apex Init Project File

## 工作流

这是一个提示词驱动的初始化 skill。你需要自己扫描用户指定范围或当前项目目录，判断哪些子文件夹缺少 `FOLDER.md`，再基于真实上下文写入内容。

对每个缺少 `FOLDER.md` 的文件夹，先阅读该文件夹内容和必要的相邻文件，理解它在项目里的真实作用。然后由你根据实际上下文创建 `FOLDER.md`，不要只根据文件夹名猜测。

只有在目录很多、需要先生成候选清单时，才可以把 `scripts/init_agents_md.py <project-root>` 当作可选辅助工具使用。这个脚本只负责列出候选目录，不负责创建文件，也不负责生成文档内容。

如果用户明确要求“重新生成 / 重跑 / 覆盖 / 刷新 / regenerate”，就进入重新生成模式：已有 `FOLDER.md` 的目录也必须重新处理，不能因为文件已存在就跳过。使用辅助脚本时传入 `--all`、`--force` 或 `--regenerate` 获取全量目录清单。

每个新建的 `FOLDER.md` 必须控制在 8 行以内：

1. `# [folder-name]`
2. `Scope: [一句话说明本目录真实职责、边界和上下游作用]`
3. `Key files: [主要入口/规则/适配器/测试位置，最多 5 个]`
4. `Rules: [本目录最重要的 1-3 条非显而易见约束]`
5. `Verify: [本目录最窄可用测试/类型检查/构建命令；无则 none]`
6. `Agents: 仅当目录职责、规则、关键入口或验证方式变化时更新本文件。`

默认模式下只处理确实缺少 `FOLDER.md` 的文件夹。重新生成模式下，按用户要求重写已有 `FOLDER.md`。
